# BYD hardware modules already stubbed (`android/hardware/bydauto/*`)

Catalog of what's already available, generated by reading the layer-1 stub
method signatures directly (not from BYD documentation — none is public).
Check here before assuming a signal needs a brand-new module; also check
which of these already have a layer-2 `com.overdrive.app.byd.<module>`
manager wired into `BydEventDaemon` (only `radar` and `bodywork` are
confirmed wired as of this writing — treat any other module below as
**stub exists, manager may not**, and verify with `grep -rn "new
XManager(" app/src/main/java/com/overdrive/app/byd/BydEventDaemon.java`).

Common base classes shared by every module (`android/hardware/bydauto/`,
not module-specific):
- `AbsBYDAutoDevice` — base `get(int,int)` / `set(int,int,int)` / `registerListener` surface most device stubs extend
- `BYDAutoDeviceManager` — top-level manager stub, `getInstance(Context)`
- `BYDAutoEventValue`, `BYDAutoFeatureIds` — shared constant/event-id stubs referenced across modules

To regenerate or extend this table after adding new stub methods:
```bash
for d in app/src/main/java/android/hardware/bydauto/*/; do
  echo "--- $(basename "$d") ---"
  grep -h "public .*(" "$d"*.java | sed 's/^[[:space:]]*//'
done
```
